74 Search a 2D Matrix

Description

Write an efficient algorithm that searches for a value in an m x n matrix.
This matrix has the following properties:

  • Integers in each row are sorted from left to right.
  • The first integer of each row is greater than the last integer of the previous row.

Example 1:

**Input:** matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3
**Output:** true

Example 2:

**Input:** matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 13
**Output:** false

Constraints:

  • m == matrix.length
  • n == matrix[i].length
  • 1 <= m, n <= 100
  • -10 4 <= matrix[i][j], target <= 104

Binary Search: map 2D array onto 1D array

  • low = 0, high = nums.length * nums[0].length - 1
  • mid = (low + high) >> 1
  • low <= high => mid ~ [0, nums.length * nums[0].length - 1]
  • mapping of 1D array onto 2D
  • row = mid / nums.length
  • col = mid % nums[0].length

Tow pointer

Concept:- If we look carefully the matrix is row-wise sorted as well as column-wise sorted. So lets start searching from the top right element of the matrix. If it matches with target return true. If it is less than target then we know it can never be found in left side of the current element in current row( as it is row-wise sorted ) and hence in can be found in the next row cause all the elements in downward direction of the current element are greater( as it is column wise sorted ). Similary if the current element is greater than the target then we need go backward in the current row cause all the elements in the back are lesser than the current element.
